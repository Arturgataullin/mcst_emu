## Описание

Проект состоит из двух основных частей:

- **Assembler** — читает текстовую программу на упрощённом ассемблере и преобразует её в бинарное представление инструкций.
- **Emulator** — загружает бинарный файл с инструкциями, исполняет их и выводит итоговое состояние регистров.

Процессор и формат инструкций находятся в файле docs/PROC.md

---

## Описание основных модулей

### Assembler

#### `lexer`
Модуль лексического анализа. Читает исходный текст программы посимвольно и преобразует его в последовательность токенов: инструкции, регистры, числовые литералы, переводы строк и конец файла. На этом этапе выполняется первичная проверка корректности входного текста.

#### `parser`
Модуль синтаксического анализа. Получает последовательность токенов от лексера и проверяет, что они образуют корректные инструкции поддерживаемого языка ассемблера. На выходе формирует внутреннее представление программы, удобное для дальнейшего кодирования.

#### `encoder`
Модуль кодирования инструкций. Преобразует внутреннее представление программы в последовательность байт в соответствии со спецификацией ISA.

#### `assembler`
Объединяет все этапы обработки входной программы: чтение файла, запуск лексического анализа, синтаксического разбора, кодирования и запись результата в бинарный выходной файл.

### Emulator

#### `memory`
Отвечает за загрузку бинарного файла с машинными инструкциями. Используется как источник инструкций для эмулятора.

#### `emulator`
Содержит модель состояния процессора, включая 16 32-битных регистров общего назначения, выполняет цикл чтения, декодирования и исполнения инструкций и в конце работы выводит итоговое состояние регистров.

## Пример ассемблирования

Для входного файла `in.s`:

```asm
LI R0 0x1
LI R1 0x8000
ADD R1 R1 R0
```

ассемблер должен сформировать бинарный файл `out.o` со следующими байтами:

```text
0x00 0x00 0x01 0x00
0x00 0x01 0x00 0x80
0x02 0x01 0x01 0x00
```

---

## Пример выполнения

Для программы:

```asm
LI R0 0x1
LI R1 0x8000
ADD R1 R1 R0
```

после исполнения ожидается состояние регистров:

```text
R0=0x1, R1=0x8001, R2=0x0, ...
```

## Сборка проекта

### Обычная сборка

```bash
mkdir build
cmake -S. -B build
cmake --build build/
```

### Сборка с тестами

```bash
mkdir -p build
cmake -S. -B build -DBUILD_TESTING=ON
cmake --build build/
```

## Запуск

### Ассемблер

Ассемблер принимает текстовый файл с программой и создаёт бинарный файл с инструкциями.

Пример запуска:

```bash
./build/assembler/assembler in.s out.o
```

### Эмулятор

Эмулятор принимает бинарный файл с инструкциями, исполняет его и печатает значения регистров.

Пример запуска:

```bash
./build/emulator/emulator out.o
```

## Тестирование

Для проекта используются автотесты на базе **Catch2**.
Запуск всех тестов:

```bash
cd build
ctest
```
