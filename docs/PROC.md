# Модель процессора

## Регистры

Процессор имеет 16 целочисленных 32-разрядных регистров общего назначения:
`R0` ... `R15`. При запуске программы все пользовательские регистры
инициализируются нулём.

Стек описывается двумя 32-разрядными статусными регистрами:

- `SP_TOP` (`SCR1`) — нижняя граница выделенной области стека;
- `SP_SIZE` (`SCR2`) — количество байт, оставшихся доступными для выделения.

Стек растёт к меньшим адресам. Если исходное значение `SP_TOP` равно `B`, то
выделенная область представляется полуинтервалом `[SP_TOP, B)`. Пустой стек имеет
`SP_TOP == B`; после выделения одного байта `SP_TOP == B - 1`.

Отрицательные значения представлены в дополнительном коде. Результаты
арифметических операций вычисляются по модулю `2^32`: биты, выходящие за
пределы 32-разрядного регистра, отбрасываются.

## Формат инструкций

Каждая машинная инструкция занимает 4 байта. Первый байт содержит opcode.
Остальные байты интерпретируются в зависимости от формата инструкции.

### Формат RI

```text
opcode a I0 I1
```

- `a` - номер регистра назначения;
- `I0` - младший байт 16-разрядного литерала;
- `I1` - старший байт 16-разрядного литерала.

### Формат RRR

```text
opcode a b c
```

- `a` - номер регистра назначения;
- `b` - номер первого регистра-источника;
- `c` - номер второго регистра-источника.

### Формат RRI8

```text
opcode a b I
```

- `a` - номер первого регистра;
- `b` - номер второго регистра;
- `I` - 8-разрядный беззнаковый литерал.

Этот формат используют инструкции доступа к памяти и преобразования `SXT`,
`BSWAP`. В ассемблере третий операнд записывается как обычный
шестнадцатеричный литерал, но должен помещаться в один байт:
`0x0` ... `0xff`.

## Модель памяти

Память адресуется байтами. Регистры и машинные слова имеют размер 32 бита,
но адрес `N` всегда указывает на конкретный байт, а не на 32-битную ячейку.

Многобайтовые операции используют little-endian порядок:

- младший байт значения хранится по меньшему адресу;
- `STH` записывает два младших байта регистра;
- `STW` записывает все четыре байта регистра;
- `LDH` и `LDW` собирают результат из байтов в том же порядке.

Например, `STW` значения `0x11223344` по адресу `0x40` записывает:

```text
[0x40] = 0x44
[0x41] = 0x33
[0x42] = 0x22
[0x43] = 0x11
```

Размер моделируемой RAM задаётся при запуске эмулятора. Реализация хранит
память разреженно: адресное пространство делится на блоки по 4 КБ, а блок
создаётся только при записи или загрузке данных в него. Чтение из ещё не
созданного блока возвращает нули. Эмулятор может дополнительно предупреждать
о чтении неинициализированных байтов опцией `--warn=uninit-ram`.

## Набор машинных инструкций

### Загрузка литералов

| Opcode | Инструкция | Семантика |
|---|---|---|
| `0x00` | `LI Ra I` | `R[a] = I` |
| `0x01` | `LUI Ra I` | `R[a] = (R[a] & 0xffff) \| (I << 16)` |

`LI` записывает 16-разрядный литерал в младшую часть регистра и заполняет
старшие 16 бит нулями. `LUI` сохраняет младшую половину регистра и заменяет
старшую половину литералом.

Пример:

```asm
LI R1 0x5678
LUI R1 0x1234
```

После выполнения `R1 = 0x12345678`. Инструкции кодируются так:

```text
0x00 0x01 0x78 0x56
0x01 0x01 0x34 0x12
```

### Арифметика, логика и сдвиги

| Opcode | Инструкция | Семантика |
|---|---|---|
| `0x02` | `ADD Ra Rb Rc` | `R[a] = R[b] + R[c]` |
| `0x03` | `SUB Ra Rb Rc` | `R[a] = R[b] - R[c]` |
| `0x04` | `AND Ra Rb Rc` | `R[a] = R[b] & R[c]` |
| `0x05` | `OR Ra Rb Rc` | `R[a] = R[b] \| R[c]` |
| `0x06` | `XOR Ra Rb Rc` | `R[a] = R[b] ^ R[c]` |
| `0x07` | `SLL Ra Rb Rc` | `R[a] = R[b] << (R[c] & 0x1f)` |
| `0x08` | `SRL Ra Rb Rc` | `R[a] = R[b] >> (R[c] & 0x1f)` |
| `0x09` | `SRA Ra Rb Rc` | арифметический сдвиг `R[b]` вправо |
| `0x0a` | `MUL Ra Rb Rc` | `R[a] = R[b] * R[c]` |
| `0x0b` | `UDIV Ra Rb Rc` | беззнаковое `R[b] / R[c]` |
| `0x0c` | `SDIV Ra Rb Rc` | знаковое `R[b] / R[c]` |

Для `SLL`, `SRL` и `SRA` величина сдвига маскируется значением `0x1f`.
`SRA` заполняет освободившиеся старшие биты прежним знаковым битом.

Целочисленное деление усекает результат к нулю. Деление на ноль завершает
работу эмулятора с сообщением об ошибке.

### Доступ к памяти

| Opcode | Инструкция | Семантика |
|---|---|---|
| `0x10` | `LDB Ra Rb I` | `R[a] = zero_extend8(M[R[b] + I])` |
| `0x11` | `LDH Ra Rb I` | `R[a] = zero_extend16(M[R[b] + I ... R[b] + I + 1])` |
| `0x12` | `LDW Ra Rb I` | `R[a] = M[R[b] + I ... R[b] + I + 3]` |
| `0x13` | `STB Ra Rb I` | `M[R[b] + I] = low8(R[a])` |
| `0x14` | `STH Ra Rb I` | `M[R[b] + I ... R[b] + I + 1] = low16(R[a])` |
| `0x15` | `STW Ra Rb I` | `M[R[b] + I ... R[b] + I + 3] = R[a]` |

Для всех инструкций этого блока эффективный адрес вычисляется как
`R[b] + I`, где `I` - 8-разрядное беззнаковое смещение. Доступ не обязан быть
выровнен: например, `STW Ra Rb 0x1` может изменить байты в двух соседних
внутренних 32-битных ячейках памяти.

### Преобразования

| Opcode | Инструкция | Семантика |
|---|---|---|
| `0x16` | `SXT Ra Rb I` | знаковое расширение `R[b]` по режиму `I` |
| `0x17` | `BSWAP Ra Rb I` | перестановка байтов `R[b]` по режиму `I` |

Режимы `SXT`:

- `0x0` - знаково расширить младший байт;
- `0x1` - знаково расширить младшее полуслово;
- остальные значения - оставить исходное значение без изменений.

Режимы `BSWAP`:

- `0x1` - поменять местами два младших байта, старшие 16 бит сохранить;
- `0x2` - поменять порядок всех четырёх байтов слова;
- остальные значения - оставить исходное значение без изменений.

### Статусные регистры и стек

| Opcode | Инструкция | Кодировка | Семантика |
|---|---|---|---|
| `0x20` | `SCRW SCR Rb` | `0x20 SCRi b 0x00` | `SCR = R[b]` |
| `0x21` | `SCRR Ra SCR` | `0x21 a SCRi 0x00` | `R[a] = SCR` |
| `0x22` | `ASPI Ra I` | `0x22 a I0 I1` | изменить `SP_TOP` на знаковое `imm16` |
| `0x23` | `ASPR Ra Rb` | `0x23 a b 0x00` | изменить `SP_TOP` на знаковое значение `R[b]` |

Отрицательное смещение выделяет память, положительное освобождает. Сначала
проверяется доступность диапазона, затем одновременно обновляются `SP_TOP` и
`SP_SIZE`, а новое значение `SP_TOP` записывается в `R[a]`. Для выделения `N`
байт выполняется `ASPI Ra 0x10000-N`; возвращённый адрес является первым байтом
выделенного диапазона.

Эмулятор завершает выполнение с ошибкой `stack overflow`, если запрошено больше
байт, чем содержит `SP_SIZE`, и с `stack underflow`, если освобождение переносит
`SP_TOP` выше исходной границы стека. Запись `SCRW SP_TOP` задаёт эту исходную
границу.

Опция `--clear-stack` заполняет освобождаемый диапазон `[old SP_TOP, new SP_TOP)`
нулями. Независимо от этой опции освобождённые байты снова помечаются
неинициализированными. Поэтому сочетание `--clear-stack --warn=uninit-ram`
возвращает нули при последующем чтении и одновременно выводит предупреждение.
Если включён режим `--trace=ram-wr`, обнуление выводится как запись RAM на такте
освобождающей команды `ASPI` или `ASPR`.

## Псевдокоманды ассемблера

Псевдокоманды не имеют собственных opcode. Ассемблер раскрывает их в
последовательности машинных инструкций до этапа кодирования.

### `MOV Ra Rb`

```text
OR Ra Rb Rb
```

Результат: `R[a] = R[b]`.

### `NEG Ra Rb`

```text
LI R31 0x0000
SUB Ra R31 Rb
```

Результат: `R[a] = -R[b]`.

### `NOT Ra`

```text
LI R31 0xffff
LUI R31 0xffff
XOR Ra Ra R31
```

Результат: `R[a] = ~R[a]`.

### `LFI Ra L`

```text
LI Ra low16(L)
LUI Ra high16(L)
```

Результат: в `R[a]` записывается полный 32-разрядный литерал.

`R31` используется реализацией как скрытый временный регистр ассемблера.
Он недоступен в исходном языке (`R0` ... `R15`) и не выводится среди
пользовательских регистров.

## Формат исходной программы

- одна инструкция размещается на одной строке;
- регистр записывается как `R0` ... `R15`;
- числовые литералы задаются только в шестнадцатеричном виде с префиксом
  `0x` или `0X`;
- `LI` и `LUI` принимают значения `0x0` ... `0xffff`;
- `LDB`, `LDH`, `LDW`, `STB`, `STH`, `STW`, `SXT` и `BSWAP` принимают
  третий операнд `0x0` ... `0xff`;
- `SCRW` и `SCRR` принимают только `SP_TOP` или `SP_SIZE` в позиции SCR;
- `ASPI` принимает 16-разрядный битовый шаблон `0x0` ... `0xffff`, который
  при исполнении интерпретируется как знаковое значение;
- `ASPR` интерпретирует содержимое регистра-источника как знаковое 32-разрядное
  значение;
- `LFI` принимает значения `0x0` ... `0xffffffff`;
- текст после `#` считается комментарием.

Пример:

```asm
LFI R0 0x12345678
MOV R1 R0
NEG R2 R1
NOT R2
```
