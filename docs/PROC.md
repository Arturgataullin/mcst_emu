## Модель процессора

### Регистры

Процессор имеет:

- `16` регистров общего назначения;
- размер каждого регистра — `32 бита`;
- имена регистров: `R0` ... `R15`;
- в начале работы все регистры инициализируются нулём.

### Инструкции

- размер одной инструкции — `4 байта`.

Поддерживаются две инструкции.

---

## Набор инструкций

### `LI` — Load Immediate

Загружает 16-битный литерал в регистр.

**Семантика:**

```text
R[a] = I = I0 | (I1 << 8)
```

**Текстовый формат:**

```text
LI Ra I
```

**Бинарный формат:**

```text
0x00 a I0 I1
```

Где:

- `a` — номер регистра назначения;
- `I0` — младший байт литерала;
- `I1` — старший байт литерала.

Пример:

```text
LI R1 0x8000
```

кодируется как:

```text
0x00 0x01 0x00 0x80
```

---

### `ADD` — Add

Складывает значения двух регистров и записывает результат в регистр назначения.

**Семантика:**

```text
R[a] = R[b] + R[c]
```

**Текстовый формат:**

```text
ADD Ra Rb Rc
```

**Бинарный формат:**

```text
0x02 a b c
```

Где:

- `a` — номер регистра назначения;
- `b` — номер первого операнда;
- `c` — номер второго операнда.

---

## Формат входной программы

Входная программа — это текстовый файл, в котором каждая инструкция находится на отдельной строке.

Пример:

```asm
LI R0 0x1
LI R1 0x8000
ADD R1 R1 R0
```
