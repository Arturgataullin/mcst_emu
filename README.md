# Эмулятор процессора и ассемблер

Проект содержит ассемблер и эмулятор простого 32-разрядного процессора.
Процессор имеет 16 регистров общего назначения `R0` ... `R15`, а каждая
машинная инструкция занимает 4 байта.

Подробное описание регистров, бинарных форматов и семантики инструкций
находится в [docs/PROC.md](docs/PROC.md).

## Возможности

Ассемблер поддерживает:

- загрузку литералов: `LI`, `LUI`;
- арифметику: `ADD`, `SUB`, `MUL`, `UDIV`, `SDIV`;
- поразрядные операции: `AND`, `OR`, `XOR`;
- сдвиги: `SLL`, `SRL`, `SRA`;
- псевдокоманды: `MOV`, `NEG`, `NOT`, `LFI`;
- шестнадцатеричные литералы с префиксом `0x` или `0X`;
- комментарии, начинающиеся с `#`.

Эмулятор выполняет машинный код, проверяет индексы регистров и размер
программы, сообщает об ошибке при делении на ноль и выводит итоговое состояние
регистров.

## Структура проекта

- `common` - общие определения ISA и opcode;
- `assembler` - lexer, parser, lowering псевдокоманд, encoder и binary writer;
- `emulator` - память команд и цикл fetch/decode/execute;
- `tests` - модульные, интеграционные и сквозные тесты;
- `docs/PROC.md` - спецификация процессора и системы команд;
- `examples` - [примеры программ](examples/README.md) на ассемблере.

## Пример

Файл `in.s`:

```asm
LFI R0 0x12345678
MOV R1 R0
NEG R2 R1
NOT R2
```

Псевдокоманда `LFI` раскрывается в `LI` и `LUI`, а остальные псевдокоманды
транслируются в существующие арифметические и логические инструкции.

## Требования

- компилятор с поддержкой C++20;
- CMake 3.20 или новее;
- GCC/Linux для целевого окружения задания;
- доступ к GitHub при первой тестовой сборке: Catch2 загружается через
  `FetchContent`.

## Сборка

Обычная сборка без тестов:

```bash
cmake -S . -B build
cmake --build build
```

Сборка с тестами:

```bash
cmake -S . -B build -DBUILD_TESTING=ON
cmake --build build
```

## Запуск

Ассемблирование программы:

```bash
./build/assembler/assembler in.s out.o
```

Выполнение бинарного файла:

```bash
./build/emulator/emulator out.o
```

При успешном выполнении эмулятор печатает все пользовательские регистры:

```text
R0=0x12345678, R1=0x12345678, R2=0x12345677, ...
```

## Тестирование

Запуск всего набора тестов:

```bash
ctest --test-dir build --output-on-failure
```

Набор включает:

- unit-тесты lexer, parser и encoder;
- интеграционные тесты ассемблера;
- тесты памяти и выполнения инструкций эмулятором;
- сквозные тесты `исходный код -> ассемблер -> бинарный файл -> эмулятор`.

## Трассировка исполнения

Трасса команд включается опцией `--trace=disasm`:

```bash
./build/emulator/emulator --trace=disasm out.o
```

Нумерация тактов начинается с `0`: номер такта равен количеству команд,
исполненных перед текущей командой.

Вывод можно ограничить включительными диапазонами тактов; пересекающиеся
диапазоны допускаются:

```bash
./build/emulator/emulator --trace=disasm --trace-ticks=1,3-7,5-10 out.o
```

Поддержка трассировки включена при сборке по умолчанию. Отключить её можно
через `cmake -S . -B build -DTRACING=0`.

## Ограничения

- поддерживаются только шестнадцатеричные числовые литералы;
- одна исходная инструкция должна находиться на одной строке;
- `NEG` и `NOT` используют скрытый временный регистр `R31` при раскрытии.
  Это соглашение реализации описано в `docs/PROC.md`.
